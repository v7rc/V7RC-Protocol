# V7RC IO Command Protocol 說明

因應更多的應用需求，V7RC 開放的 Protocol 已進行了修正以達到更好的相容性（支援度可能會因不同的開發板與韌體版本而有所不同）[1]。

## 基本規範

因應基本藍牙 BLE 的傳輸 Packet 長度限制，命令長度統一限制在 **20 Bytes 以內**[1]。
目前支援的規格主要為 20 Bytes（16 Bytes 版本在新版 APP 中已不再支援）[1]。

* **命令格式**：前三碼統一表示「命令代碼」[1]。
* **結尾符號**：統一以 `#` 表示[1]。
* **空白補齊**：沒有用到的欄位統一以 `0` 補滿（`CMD` 指令則以空白符號補足）[1, 2]。

## 控制指令詳細說明

### A. SRV (基本 PWM 控制)
* **說明**：標準 PWM 控制，長度修改為 20 Bytes[1]。
* **範例**：
  * `SRV1500150015001500#` -> 支援 P0, P1, P2 的 PWM 訊號 (Microbit 目前標準版只有支援三個 PWM)[1, 3]。
  * `SRV1500100018002000#` -> 支援 C1, C2, C3, C4 的 PWM 訊號[1, 4]。
* **注意**：16 Bytes 版本如 `SRV150015001500#` (P0-P2訊號) 由於新 APP 不再支援，請避免使用[1]。

### B. SR2 (第二組 PWM 控制)
* **說明**：若開發板提供超過 4 個 PWM 控制伺服器，此組命令用來控制 C5-C8 的伺服馬達[3]。
* **範例**：`SR21500100018002000#` -> 對應 C5, C6, C7, C8 的 PWM 訊號[3]。

### C. SS8 (簡易 PWM，可同時使用 8 個 PWM)
* **說明**：將每個 PWM 控制簡化為 `00`-`FF` 的十六進位數值（精細度會降低）。運算方式為：取得數字後轉為 10 進位，並直接乘以 10 成為 PWM 數值[3]。
* **範例**：`SS896969696969696#` -> `96` 轉為 10 進位是 150，乘以 10 後數值為 1500[3]。

### D. SSD (簡易控制裝置加上按鈕控制) *(未實作)*
* **說明**：每組 PWM 簡化為三碼表示角度，最後四碼表示四個 GPIO 信號（以 `0` 或 `1` 表示）[3]。
* **範例**：`SSD0901801800001100#`[3]
  * 表示角度：C1為 90度, C2為 180度, C3為 180度, C4為 0度[4]。
  * GPIO 狀態：GPIO 0 (ON), GPIO 1 (ON), GPIO 2 (OFF), GPIO 3 (OFF)[4]。

### E. SRT (基本 PWM 的變形 - Tank 控制)
* **說明**：由基本 PWM 變形而來，CH1 與 CH2 會在韌體端被轉換成 Tank 控制的訊號[4]。
* **範例**：`SRT1500150015001500#`[4]。

### F. SST (簡易控制裝置的變形 - Tank 控制) *(未實作)*
* **說明**：由簡易控制裝置變形而來，CH1 與 CH2 會在韌體端轉換成 Tank 控制的訊號[4]。
* **範例**：`SST0901801800001100#` (角度與 GPIO 表示法同 SSD 指令)[4]。

## 燈光控制指令 (搭配 WS2812 晶片 LED 燈使用)

每顆 LED 燈都以 `RGBM` 四個變數來表示：
* **R (紅色)**：以 `0`-`F` 表示[5]。
* **G (綠色)**：以 `0`-`F` 表示[5]。
* **B (藍色)**：以 `0`-`F` 表示[5]。
* **M (閃爍動作)**：以每秒為單位。`0` 表示不亮，`1` 表示每秒有 100ms 會亮，以此類推。如果是 `A` 以上，表示恆亮[5]。
  *(例如：`F00A` 表示紅燈恆亮，`FFF5` 表示白燈每秒亮 500ms、暗 500ms)[5]*。

### G. LED (第一組四顆 LED 燈)
* **格式**：20 Bytes。
* **範例**：`LED0111100000111110000022222#` -> 控制前方的 LED[5]。

### H. LE2 (第二組四顆 LED 燈)
* **格式**：20 Bytes。
* **範例**：`LE20111100000111110000022222#` -> 控制第二組的 LED[2, 5]。

### I. LE? (第 N 組四顆 LED 燈)
* **說明**：`?` 代表數字 `N`，用來控制第 N 組的四顆 LED 燈[2]。
* **範例**：`LE50111100000111110000022222#` -> 代表第五組的 LED[2]。

## 裝置自訂指令

### J. CMD (傳遞給 Device 的控制命令)
* **說明**：可將命令傳遞給 DEVICE 端進行控制，完全由 DEVICE 端開發者自行定義。最大長度為 16 字元（不含前綴與結尾號）。命令長度若不滿 16 個字元，需使用**空白符號**補足[2]。
* **範例**：若指令為 `RESET`，完整封包為：`CMDRESET          #`[2, 6]。
